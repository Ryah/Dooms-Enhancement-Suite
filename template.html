Expressions
DES handles expressions

You do not need SillyTavern's separate Expressions extension. With Per-Character Expressions on, DES classifies each character's mood every turn and swaps the matching sprite onto their portrait card.

How to add sprites for a character
  1. Open the character in Character Workshop → Appearance tab.
  2. Click Open expression folder.
  3. Drop image files directly into that folder (not into a sub-folder). .png and .webp both work. Name them after the emotion the classifier outputs: joy, sadness, annoyance, optimism, neutral, love, nervousness, relief, remorse, realization, surprise, etc. — e.g. joy.webp, sadness.png.
  4. Reload SillyTavern. The next AI message will start swapping sprites automatically.

Turn on Show Expression in Tooltip below to see which label was picked when you hover a portrait.

Countdown Length Once the threshold is reached, a visible countdown begins at this number. Each low-tension response ticks it down (very low tension ticks faster). When it hits zero, you'll be prompted to pick a plot twist.

Trap Mode Silent mode — hides the countdown and debug badge from view. When the counter triggers, it generates one twist and injects it without showing you what it picked. You won't know it's coming.
Speed Rules
The countdown ticks faster when tension is very low:
Tension 1 → countdown ticks by 3
Tension 2 → countdown ticks by 2
Tension 3-4 → countdown ticks by 1
Tension 5+ → resets streak (story is tense enough)

Identity

Name is read-only. Click a relationship chip to set an override — it wins over the AI's per-turn classification and is included when you Inject into Scene. Click the selected chip again to clear.

Portrait

Shown on the portrait bar. The preview at left updates as soon as you pick a file.

Dialogue color

Used for this character's dialogue in chat and the name label on the portrait card. Pick from the palette or click Custom… for a precise color.

Expressions

Expression sprites live in data/default-user/characters/<name>/. Open the folder to add images named after the classifier's emotion labels — e.g. joy.webp, sadness.png, annoyance.webp, optimism.png, neutral.webp, realization.png, surprise.webp, etc. Both .png and .webp are supported. DES's expression sync will pick them up automatically.

Description (bio + appearance)

Optional context the AI sees when you click Inject into Scene. Cover physical appearance, role, manner of speaking, anything that helps the AI portray them consistently. Plain text — be specific about what they look like since most models won't actually see the portrait image.

Attached lorebook

Pick a SillyTavern lorebook (World Info) to activate when this character is injected. Type to filter; leave blank to skip.

    Injection prompt (advanced)

    The scene-direction text sent to the AI when you click Inject into Scene. Leave blank to use the default. Supports variables {name}, {description}, {lorebook}, {relationship}, and optional blocks {?var}…{/var} that are only included when the variable is filled in.

    Attach portrait to message

    When on, clicking Inject into Scene also attaches this character's portrait to your next user message so vision-capable models (Claude 3.x, GPT-4o, Gemini, etc.) can actually see them. Silently no-ops on text-only models — the description above is your reliable channel either way.

    Off by default. Setting is global, not per-character.

    Banish from generations

    When on, every AI generation receives a standing direction telling it that this character is not in the scene and must not appear — no dialogue, no actions, no presence. Use when a character keeps reappearing turn after turn even after Eject. Also soft-removes them from the panel. Toggle off to allow them back in.

    Standing ban — persists across turns and reloads until you toggle it off.

    Stuck injection?

    Inject into Scene is a one-shot direction — it should clear after the AI's next reply. If a character keeps getting referenced every turn anyway, force-wipe Doom's injection slot here. Your stored description, lorebook attachment, and prompt template are kept; only the live pending injection is cleared.
